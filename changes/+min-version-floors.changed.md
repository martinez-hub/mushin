Raised minimum dependency floors to a tested minimum and added a CI job that runs the suite against the lowest declared versions, so the floors stay honest. The previous floors did not actually work: NumPy 2 is ABI-incompatible with torch built against NumPy 1.x (PyTorch ships NumPy-2 wheels from 2.4.0), and Lightning's 2.0 rewrite dropped the 1.5 API. New floors: pytorch-lightning >= 2.0, torch >= 2.4 (non-Intel; Intel-macOS stays 2.2.x with NumPy 1.x).
